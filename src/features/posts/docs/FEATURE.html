Post Creation Technical Journey

A post records a public incident with evidence, time, location, and ministry board classification. The home route composes the existing map canvas with a post creation overlay; post logic stays in the posts feature.

Data Interface

Location hierarchy

country is required. state and district are optional, and district cannot be selected without state.

Board derivation

Country-only posts use Union ministries. Any selected state or union territory uses a state-level department list for that jurisdiction, with or without district.

Post table

The SQLite table stores title, description, board, evidence URI, evidence type, country, nullable state, nullable district, timestamp, and audit timestamps.

Evidence

Links are stored directly. File evidence is written to data/uploads/post-evidence and stored as a generated evidence URI.

User Interface

A fixed rounded bordered + button is centered near the bottom of the viewport. It opens a bordered right-side pane with a labeled form, country/state/district selectors, derived board selector, evidence input, and side-by-side date and time controls. The file evidence control presents a visible choose-file segment, a divider, and a filename display segment while the underlying native file input remains accessible and handles form submission. The date control opens a shadcn calendar popover; the time control defaults to the current clock time.

Tests

Coverage includes location-to-board resolution, district validation, schema validation, Drizzle persistence, evidence upload cleanup, evidence route validation, and post creation pane behavior.

Known Gaps

The constants module provides a board list for every Indian state and union territory so the form and validation work end-to-end. Union ministries are source-marked as verified; most state-level lists currently use a standard department taxonomy with incomplete source metadata until each jurisdiction is verified against official government sources.
